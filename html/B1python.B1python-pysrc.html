24  General concepts: 
25   
26      As it was already said, these functions are based on Matlab(R) scripts. 
27          It was kept iln mind therefore to retain Compatibility to the Matlab(R) 
28          version more or less. However, we are in Python and it would be 
29          foolish not to use the possibilities and tools it provides. In the 
30          next lines I note the differences. 
31       
32      the numbering of the pixels of the detector starts from 1, in case of the 
33          beam coordinates, thus the values from the Matlab(R)-style 
34          intnorm*.log files are usable without modification. On contrary to 
35          Matlab(R), Python counts the indices from 0, so to get the real value 
36          of the beam coordinates, one should look at the pixel bcx-1,bcy-1. All 
37          functions in this file which need the beam coordinates as input or 
38          supply them as output, handle this automatically. 
39       
40      the radially averaged datasets are in a data dictionary with fields 'q', 
41          'Intensity', 'Error' and possible 'Area'. The approach is similar to 
42          that of the Matlab(R) version, however in Python, dictionary is the 
43          best suited container. 
44           
45      the mask matrices and corrected 2D data are saved to and loaded from mat 
46          files. 
47           
48      if a function classifies measurements depending on energies, it uses 
49          always the uncalibrated (apparent) energies, which were set up in the 
50          measurement program at the beamline. 
51           
52      the difference between header and param structures (dictionaries in 
53          Python) are planned to be completely removed. In the Matlab(R) version 
54          the fields of a header structure consist a subset of the ones of the 
55          param structure. When the data evaluation routines get implemented, 
56          the header dictionary extracted from the input datasets will get 
57          extended during the evaluation run by newly calculated values.
