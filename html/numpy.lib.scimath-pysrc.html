 36

 46      Examples 
 47      -------- 
 48   
 49      First, consider an input of type short: 
 50   
 51      >>> a = np.array([1,2,3],np.short) 
 52   
 53      >>> ac = np.lib.scimath._tocomplex(a); ac 
 54      array([ 1.+0.j,  2.+0.j,  3.+0.j], dtype=complex64) 
 55   
 56      >>> ac.dtype

 58   
 59      If the input is of type double, the output is correspondingly of the 
 60      complex double type as well: 
 61   
 62      >>> b = np.array([1,2,3],np.double) 
 63   
 64      >>> bc = np.lib.scimath._tocomplex(b); bc 
 65      array([ 1.+0.j,  2.+0.j,  3.+0.j]) 
 66   
 67      >>> bc.dtype 
 68      dtype('complex128') 
 69   
 70      Note that even if the input was complex to begin with, a copy is still 
 71      made, since the astype() method always copies: 
 72   
 73      >>> c = np.array([1,2,3],np.csingle) 
 74   
 75      >>> cc = np.lib.scimath._tocomplex(c); cc 
 76      array([ 1.+0.j,  2.+0.j,  3.+0.j], dtype=complex64) 
 77   
 78      >>> c *= 2; c 
 79      array([ 2.+0.j,  4.+0.j,  6.+0.j], dtype=complex64) 
 80   
 81      >>> cc 
 82      array([ 1.+0.j,  2.+0.j,  3.+0.j], dtype=complex64)
